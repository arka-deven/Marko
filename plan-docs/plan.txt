What We're Building and Why
Cialdini Institute has an execution problem, not an idea problem. Good growth ideas exist — the bottleneck is the time
and headspace required to validate, plan, and run them in parallel. This document outlines the full build plan for an
AI-powered Growth Agent that removes that bottleneck entirely: a system that generates ideas, scores them, validates
them through AI analysis, produces the marketing assets, tracks results, and kills what isn't working — continuously,
without human bottlenecks.
The system is designed to run 20–30 experiments simultaneously across all three business priorities: growing the
Cialdini Institute community, driving AI Bob subscriptions, and establishing the CIP as the industry-standard influence
assessment. This plan covers everything from the exact tech stack and architecture to the marketing playbook,
Higgsfield AI video integration, auth design decisions, pipeline flows, and 12-week delivery timeline.

Production Tech Stack
The prototype runs inside Claude.ai as a React artifact with Artifact-native persistent storage. It is technically functional
but cannot be shared, scheduled, or connected to external services. The production build migrates this to a standalone
deployable system. Every decision below is made with one constraint in mind: no BullMQ/Redis queues on Vercel. That
combination requires a persistent worker process that Vercel's 60-second serverless timeout cannot support — a
migration we've already had to do on other projects. We're building around that from day one.

Auth: Better Auth over Supabase Auth
Supabase Auth works but it's opinionated. It stores users in its own auth schema, your role logic has to live in RLS
policies that can get complex fast, and it's tightly coupled to Supabase's infrastructure. For three users right now it's fine
— but if this system ever needs to onboard external team members, agencies, or white-label access, Supabase Auth
becomes a wall.
Better Auth (better-auth.com) is a TypeScript-first auth library that runs in your own Next.js API routes, stores sessions in
your own Supabase Postgres tables, and gives you full control over the session schema, roles, and multi-tenancy. No
external auth service. No vendor lock-in beyond what you're already using. And critically — it ships with first-class
support for social login (Google, Microsoft), magic links, and org-level role management.

Pipeline Stage Detail
01 INPUT
Ideas enter the system three ways: manual entry via the dashboard (any team member), the Inngest weekly cron that
fires Monday at 6 AM ET and generates 15 ideas automatically, or a voice memo transcribed via Whisper. Each idea
carries category, target language(s), and a Cialdini principle tag.
→ Category: Content/SEO · Community · Partnerships · Paid · AI Bob · International · Events · Email · Referral
→ Language targets: EN · ES · PT · FR · DE · AR · HI · ZH · JA · KO
→ Cialdini principle: Reciprocity · Commitment · Social Proof · Authority · Liking · Scarcity · Unity
02 GENERATION ENGINE
A Next.js API route calls Claude Sonnet 4 with a structured prompt containing the business context, category
constraints, language targets, and examples of high-quality vs low-quality ideas from previous sessions (Langfuse
traces feed this). Output is a JSON array validated by Zod before any write to Supabase. Invalid schema = retry, not
silent corrupt data.
→ Output fields: title, description, category, target_languages, effort_score, impact_score, estimated_cost,
validation_method, success_metrics, cialdini_principle
→ Batch size: 10–15 ideas per cron run. Up to 75 on manual generation.
→ Zod catches: missing fields, out-of-range scores, invalid language codes, null descriptions
03 SCORING ENGINE
Every idea is automatically scored on a Priority = Impact × Ease matrix (both 1–9 scale). This score is written to
Supabase immediately and used to sort the dashboard. Scores are visible to Samir and Vasu, not Courtney.
→ Priority Score = impact_score × ease_score (1–81 range)
→ Dashboard default sort: Priority Score descending
→ Validation may revise scores — stored as priority_score_revised alongside original
04 VALIDATION ENGINE
An Inngest job fires within 24 hours of generation. It calls Claude with the idea, business context, and all existing ideas
in the pipeline (so Claude can flag duplicates). Every call is traced in Langfuse — prompt in, response out, tokens
used, latency. Results write back to Supabase and move the idea to Validated (or Killed).
→ Verdict: Proceed · Kill · Pivot (with pivot instructions)
→ Output: risk_assessment, quick_win_alternatives, revised_priority_score, validation_notes
→ Langfuse tag: category + language + verdict for cross-run analysis

05 NOTIFICATION LAYER
After validation runs, three notifications fire simultaneously via Inngest. Slack gets the weekly top-5 digest in
#growth-ops. Resend sends Courtney a formatted task list of all ideas currently in Executing stage with their delegation
notes. Supabase realtime pushes a dashboard update to any open browser sessions.
→ Slack digest: idea title, category, priority score, one-line execution summary
→ Courtney email: task title, steps, tools needed, deadline, go/no-go criteria
→ Dashboard: optimistic UI update before DB confirms via Supabase realtime
06 EXECUTION PLAN ENGINE
When Samir or Vasu approves an idea, a Claude call generates a full week-by-week execution plan. This is the most
token-intensive call in the pipeline — it produces structured output with tasks, milestones, tools required, budget line
items, delegation notes, and go/no-go checkpoints. The idea moves to Executing stage.
→ Week-by-week task breakdown with owners (Vasu / Courtney / Samir)
→ Budget breakdown per line item (maps to Stripe-tracked spend)
→ Go/no-go criteria at each milestone (used by auto-kill engine)
→ Delegation notes formatted for Courtney's copy-paste workflow
07 CONTENT ENGINE
For content-category ideas, the content engine produces publish-ready assets. Claude generates the blog post,
LinkedIn copy (2–3 A/B variants), email sequence, and passes approved text to the Higgsfield API for video
generation. Multi-language versions are produced for the top target markets. Everything is stored in Supabase pending
human approval.
→ Blog post: 800–1200 words, SEO-optimised, Cialdini principle woven in
→ LinkedIn: 2–3 variants with different hooks, same core message
→ Email sequence: 3–5 emails per idea (welcome → nurture → CTA)
→ Higgsfield: cinematic video asset generated via Cloud API (image-to-video or text-to-video)
→ Multilingual: EN → ES/PT/ZH via Claude translation + cultural adaptation pass
08 METRICS ENGINE
A daily Inngest cron pulls real conversion data from three sources and maps it to idea IDs in Supabase. ROI is
calculated automatically. No manual logging required after setup. Results feed the ROI dashboard and the auto-kill
engine.
→ Google Analytics API: pageviews, signup conversions, CIP traffic
→ Stripe API (read-only): AI Bob subscription revenue, new MRR attributed to experiments
→ Squarespace API: community page traffic, CIP completion counts
→ ROI = attributed_revenue - (idea_cost + higgsfield_spend + ad_spend)

09 AUTO-KILL ENGINE
An Inngest job checks all ideas in Executing stage daily. If a low-effort idea shows no traction after 14 days, it is
flagged for kill review. High-effort ideas get 30 days. Samir or Vasu confirm the kill. Freed budget is surfaced as a
reallocation suggestion to the top-scoring scaled winners.
→ Low-effort threshold: 14 days, no measurable traction (zero attributed conversions)
→ High-effort threshold: 30 days, below minimum ROI floor
→ Kill confirmation required from Samir or Vasu (not Courtney)
→ Reallocation: agent suggests moving freed budget to scaled experiments above ROI floor
10 REPORT & LEARNING LOOP
Monthly, Claude generates an executive report from the full Supabase dataset: aggregate ROI by category, best and
worst experiments, language market performance, and recommendations for the next month. Langfuse prompt traces
from the month are analysed to identify which prompt patterns produced the highest-ROI ideas — that analysis feeds
back into the generation prompt for the next cycle.
→ Aggregate ROI by category and language
→ Top 5 and bottom 5 experiments with full trace
→ Language market breakdown (which markets converted best)
→ AI recommendations: next month focus areas based on data
→ Prompt improvement: Langfuse traces analysed for high-ROI vs low-ROI idea patterns
